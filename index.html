// ai-assistant.js ішінде:
const GROQ_API_KEY = 'GROQ_KEY_PLACEHOLDER'; // ← нақты ключ емес!

// .github/workflows/deploy.yml ішінде:
- name: Inject API key
  run: sed -i "s|GROQ_KEY_PLACEHOLDER|${{ secrets.GROQ_API_KEY }}|g" ai-assistant.js

Менің ЖИ тәжірибем

> Қандай құрал ең көп көмектесті?

Бұл жұмысты орындауда маған GitHub Models (Llama 3 арқылы) ең көп көмектесті. HTML структурасын түсіндіруде, CSS Grid мен Flexbox айырмашылығын нақты мысалдармен көрсетуде өте пайдалы болды. Сонымен қатар Claude арқылы API интеграциясының логикасын терең түсіндім — fetch(), async/await және error handling концепцияларын.

> ЖИ нені дұрыс/ыңғайсыз жасады?

ЖИ HTML/CSS структурасын өте жылдам жасады, бірақ кейде тым күрделі шешімдер ұсынды. Мысалы, қарапайым toggle функциясы үшін 50 жолды код берді — мен оны 10 жолға қысқарттым. Нақтырақ prompt жазу арқылы (мысалы: «тек vanilla JS, минималды код») нәтиже жақсарды. CORS мәселесін ЖИ бірден анықтамады — бұны өзім зерттеп шештім.

> ЖИ 5 жылдан кейін әзірлеушіні қалай өзгертеді?

5 жылдан кейін рутиналық код жазу толық автоматтандырылады. Әзірлеушілер синтаксис жаттаудан гөрі: архитектура жобалауға, бизнес логикасын шешуге, ЖИ генерациялаған кодты тексеруге және өнімділікті оптимизациялауға фокустанады. ЖИ — бұл жаңа «стероид», ол мықты программистті одан да мықты жасайды, бірақ негізді білмейтінді алдай алмайды.
